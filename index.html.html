🚀 Prompt — Génération de la boutique Shopify complète
Parfait, le design me convient. Maintenant transforme ce design HTML en une boutique Shopify OS 2.0 complète, prête à être uploadée en ZIP. EXIGENCES DE LA BOUTIQUE : 1. Architecture OS 2.0 — JSON templates, sections Liquid avec schemas, config/settings_schema.json 2. Page d'accueil (templates/index.json) avec : - Hero produit avec image, titre accrocheur, prix, bouton d'achat - Section bénéfices / arguments clés - Témoignages clients - CTA final 3. Page produit (templates/product.json) avec : - Galerie image + infos produit (titre, prix, variantes, bouton ATC) - Sections de conversion (bénéfices, social proof, FAQ) - Sticky buy bar sur mobile 4. Tout personnalisable depuis Shopify — TOUT doit être modifiable depuis Boutique en ligne > Personnaliser : - Couleurs (principale, secondaire, fond, texte) - Polices - Textes de chaque section - Logo 5. Technique : - Aucune couleur codée en dur dans le CSS (tout via CSS variables générées depuis settings) - Images via srcset et loading="lazy" - Mobile-first, bouton CTA visible au-dessus de la ligne de flottaison - Fichier locales/fr.default.json inclus - README.md avec instructions d'installation OUTPUT : Génère tous les fichiers et crée un ZIP téléchargeable nommé boutique-[nom-produit].zip

Étape 4 — Corriger les Problèmes

Les bugs les plus courants et comment les corriger avec Claude.

Méthode universelle : Fais une capture d'écran du problème → upload dans Claude → dis ce qui ne va pas → il corrige le fichier concerné.

Problèmes fréquents et prompts de correction :

  • 🎨

    Les couleurs ne changent pas dans le customizer

    Les couleurs sont codées en dur dans le CSS au lieu d'utiliser les settings Shopify.

    Prompt de correction
    Dans ma boutique Shopify, les couleurs ne se mettent pas à jour quand je les change dans le customizer. Corrige le fichier assets/theme.css et layout/theme.liquid pour que TOUTES les couleurs soient définies via des CSS variables générées depuis {{ settings.colors_* }} dans theme.liquid. Aucune valeur hex ne doit apparaître dans theme.css.
  • 📱

    Le design est cassé sur mobile

    Textes trop grands, images débordantes, boutons trop petits.

    Prompt de correction
    Voici une capture du rendu mobile de mon thème (joint la capture). La section [NOM DE LA SECTION] est cassée : [DÉCRIS LE PROBLÈME]. Corrige le CSS dans assets/theme.css et si nécessaire le HTML dans sections/[fichier].liquid. Renvoie uniquement les fichiers modifiés.
  • 🛒

    Le bouton "Ajouter au panier" ne fonctionne pas

    Le formulaire produit ou l'ID de variant est mal configuré.

    Prompt de correction
    Le bouton "Ajouter au panier" de ma page produit ne fonctionne pas. Vérifie que le formulaire dans sections/product-hero.liquid utilise bien la syntaxe Shopify correcte : {% form 'product', product %} avec <input type="hidden" name="id" value="{{ product.selected_or_first_available_variant.id }}"> et un <button type="submit" name="add">. Corrige et renvoie le fichier.
  • 🖼️

    Les images ne s'affichent pas

    Chemins d'images incorrects ou syntaxe Liquid manquante.

    Prompt de correction
    Les images de mon thème ne s'affichent pas. Dans les fichiers Liquid concernés, remplace tous les chemins d'images statiques par la syntaxe Shopify correcte : {{ product.featured_image | image_url: width: 800 | image_tag: loading: 'lazy' }} pour les images produit, et {{ section.settings.image | image_url: width: 1200 | image_tag }} pour les images de sections. Corrige tous les fichiers concernés.
  • 📝

    Les textes ne sont pas modifiables dans le customizer

    Les textes sont codés en dur dans le Liquid au lieu d'être dans les schemas.

    Prompt de correction
    Dans la section [NOM DE LA SECTION], les textes sont codés en dur et je ne peux pas les modifier depuis le customizer Shopify. Mets à jour le fichier sections/[fichier].liquid pour que tous les textes (titres, descriptions, labels de boutons) soient des settings modifiables dans le schema JSON de la section. Chaque texte doit avoir un "type": "text" ou "richtext" dans le schema avec une valeur par défaut.
